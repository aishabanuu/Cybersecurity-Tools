# Web Application Testing

This repository contains notes, commands, examples, and lab exercises for learning web application security testing in authorized environments.

## Objectives

- Understand how web applications work
- Learn HTTP requests and responses
- Identify common web application vulnerabilities
- Practice reconnaissance and enumeration
- Build repeatable testing workflows
- Document findings and methodologies

## Tools Covered

### Burp Suite
Intercepts, analyzes, and modifies HTTP/S traffic between a browser and a web application.

### Gobuster
Discovers hidden directories, files, and virtual hosts.

### Nikto
Scans web servers for common misconfigurations and known issues.

### SQLMap
Assists in testing for SQL injection vulnerabilities.

## Learning Path

1. Learn HTTP fundamentals
2. Use Burp Suite to inspect traffic
3. Discover hidden content with Gobuster
4. Analyze server configurations with Nikto
5. Understand input validation and SQL injection concepts
6. Practice in safe lab environments

## Lab Environments

Recommended practice targets:

- DVWA (Damn Vulnerable Web Application)
- OWASP Juice Shop
- Metasploitable
- OWASP WebGoat
